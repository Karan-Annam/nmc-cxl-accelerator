### Known rough edges
- Behavioral 1-cycle SRAM — cycle counts are not latency claims (stated).
- Single outstanding command; no command queue/doorbell yet.
- MMIO map is bespoke (not PCIe config space); no LTSSM/PHY.
- Fixed-point only (Q8.8/Q16.16); no FP datapath.
- GNN control-traffic result (0.11×) awaits the command-batching fix.
- The FPGA config halves HDM to 32K words (`HDM_WORDS` in `fpga/synth.tcl`):
  the full 64K dual-read map needs 128 RAMB36 vs the xc7s50's 75, and the
  spill lands in timing-breaking async LUTRAM. Simulation runs the full map.
- LUT count (~43.8k) exceeds the xc7s50's 32.6k — timing closes in synthesis,
  but placing on this part needs LUT reduction (rx/response queues to BRAM,
  CRC sharing).

## Roadmap (by value)

1. **Command batching** — descriptor ring in HDM + one doorbell MMIO write;
   re-run the GNN suite and turn 0.11× into the headline follow-up result.
   This is the finding the perf counters explicitly point at.
2. **150 MHz routed clock** — Tier 1 (per-bank outpost registers, 2-cycle
   memory contract, all six consumers re-pipelined) lifted the routed
   ceiling from ~90 to **~112 MHz** (board ships at 105.88). The measured
   next wall is the READ-return crossing (bank DO → rotation mux → PE
   operand regs, 8.9 ns routed): needs a registered return crossbar
   (3-cycle reads) plus an issue-continuous walk so the 6.0 elems/cycle
   floor survives the extra trailing exec cycle.
3. **PCIe host link** — swap the UART flit bridge for XDMA on an M.2 Artix
   board, tunneling flits through DMA buffers (the link layer stays).
4. **DRAM-realistic memory model** — banked latency + row hits/conflicts
   behind the SG engine; add a small reorder/settle buffer; check the 8-wide
   walk still wins.
5. **Command pipelining** — overlap softmax of query q with the score pass
   of q+1 (the three attention phases use disjoint resources much of the
   time).
6. **Wider link experiments** — 256-byte flit mode (CXL 3.x style), slot
   utilization is only 38% — measure packing improvements.
7. **More workloads** — top-k selection on-device (cuts attention readback),
   CSR SpMM (multi-vector), int8 quantized paths.
8. **CI** — Verilator build + 25 tests + lint on push; results.json drift
   check (fails if measured numbers move unexpectedly).
